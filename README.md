# Injection In-Process Inspection Checker

A Streamlit dashboard for validating injection in-process inspection Excel workbooks.

Upload an inspection report, and the app automatically detects inspection sheets, extracts dimension specifications and tolerances, checks every numeric sample measurement, highlights failures, and exports CSV/Excel validation reports.

## Local Run

1. Open a terminal in this project folder.
2. Install the required Python packages:

```bash
pip install -r requirements.txt
```

3. Start the dashboard:

```bash
streamlit run app.py
```

4. Your browser will open the local dashboard. Upload your Excel inspection file from the upload box.

## Deploy to Streamlit Community Cloud

The dashboard link is generated by Streamlit Cloud after deployment. It cannot be created only from source code unless the app is deployed to a hosting platform.

Follow these steps to get a public website URL:

1. Create or log in to a GitHub account.
2. Create a new GitHub repository.
3. Upload or push all project files to that repository, including:
   - `app.py`
   - `requirements.txt`
   - `README.md`
   - `utils/`
   - `.streamlit/config.toml`
4. Go to one of these Streamlit Cloud pages:

```text
https://share.streamlit.io/
```

or

```text
https://streamlit.io/cloud
```

5. Sign in and connect your GitHub account.
6. Select your GitHub repository.
7. Select the branch you want to deploy, usually `main`.
8. Set the main file path to:

```text
app.py
```

9. Click deploy.
10. After deployment finishes, Streamlit Cloud will provide a public website link for the dashboard.

## What The Dashboard Shows

- Uploaded file name and file size
- Number of sheets detected
- Sheet names
- Total measurements checked
- PASS count
- FAIL count
- Overall result
- Filterable validation table
- PASS vs FAIL charts
- CSV and Excel report downloads

## Supported Uploads

The app is designed for `.xlsx` and `.xlsm` inspection workbooks. If you have an older `.xls` file, convert it to `.xlsx` before uploading.
